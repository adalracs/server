Adsum CMMS

Adsum CMMS

Es una Solución ideada para controlar los procesos principales de gestión de mantenimiento. La producción del software se realizó bajo altos estándares de calidad e ingeniería de software, lo que permite que los aplicativos producidos, sean de muy alta calidad facilitando además, el trabajo del usuario, pues se usaron interfaces gráficas intuitivas y soporte multi-lenguaje.

Como la producción se realizó en Colombia, los aspectos como capacitación, asesoría y soporte local, son garantías para las compañías que adquieran nuestros productos.

Modelo Tecnológico

El sistema Adsum CMMS ha sido concebido dentro del nuevo paradigma de las soluciones orientadas al WEB, que pueden ser utilizadas usando la plataforma de Internet y puede ser usado de manera convergente, por varios usuarios en ambientes de red local (intranet) y con usuarios fuera de la red local, vía Internet (extranet) o en ambientes Monousuario Stand Alone.


Componentes

El sistema Adsum CMMS soporta el almacenamiento de los datos en una de las Bases de Datos líderes del mercado, PostgreSQL www.postgresql.org. Esta poderosa herramienta, posee las características técnicas suficientes para soportar un ambiente exigente de negocios.

El diseño arquitectónico del Sistema Adsum CMMS, fue concebido en un modelo de niveles o capas, permitiendo independencia entre la manipulación de la Base de Datos, la lógica de negocios y la interfaz de usuario o presentación.

La capa de manipulación de los datos o nivel de persistencia, se encuentra dividida en dos sub-niveles.

1. El grupo de funciones que se integra a la Base de Datos, primer sub-nivel, efectúa las operaciones CRUD, Create, Read, Update y Delete con una serie de funciones especificas. Esto permite que el sistema funcione con otra "base de datos" no implementada y rápidamente, pues solo se requiere construir las funciones CRUD especificas de ese motor de Base de Datos.

2. El grupo de funciones que se integra a la funcionalidad del sistema, segundo sub-nivel, efectúa la manipulación lógica de las operaciones CRUD, con una serie de funciones de alto nivel de abstracción. Este sub-nivel es independiente del Motor de Base de Datos.
La capa de la Lógica de Negocios, se encuentra divida en dos sub-niveles:

1. El grupo de funciones donde se encuentran las funciones que efectúan todos los procesos de negocios y soporte del sistema. Este sub-nivel es el que recurre a la camada de persistencia o camada de Manipulación de Datos.

2. El grupo de funciones que se integra con las interfaces de usuario y otros sistemas externos, conocido como el nivel de API´s, Aplication Program Interface.

La capa de presentación es desarrollada en HTML, para garantizar su operación eficiente en los navegadores Browsers disponibles en el mercado. En este nivel sólo se maneja la presentación de la información, no se gestiona nada de la lógica de negocios o de datos; también incluye una herramienta basada en XML, Extended Markup Lenguaje, para realizar funciones de Extracción, Transformación y Transporte de Datos hacia o desde otros sistemas.

Lenguajes de Desarrollo

Adsum CMMS está desarrollado bajo plataforma WEB. Para este tipo de soluciones se utilizó el lenguaje estándar de programación en Internet HTML y para garantizar la interacción con los datos, se utilizó PHP 4.x.

Plataforma de Hardware y Software Operativo:

· Windows 95, 98, 2000, NT y XP
· Todas las distribuciones de Linux con ambiente gráfico
· SCO Unix, AIX, HP/UX y Solaris

Requerimientos

Servidor

1. 512 Megas en RAM
2. 100 Megas en Disco Duro (sólo para la instalación - espacio adicional para los documentos no se incluye)
Software Servidor WEB que soporte protocolo HTTP 1.0. En la actualidad, todo programa Servidor WEB cumple con ésta especificación.Conexión a la Red (si va a ser usado al interior de la empresa, debe estar conectado a la red interna; si va a ser usado en la versión personal, no requiere conectividad).
3. Sistema Operativo: Windows 98, 2000 Server, NT, Linux, Solaris y SCO - y su configuración le permite correr con seguridad en cualquier plataforma Unix o Windows existente.


Usuarios

Un usuario sólo necesita un Navegador de Internet compatible con JavaScript 1.2, Java VM y HTML 4.0. El explorador que actualmente cumple completamente éstas características, es el Microsoft Internet Explorer 5.0, gratuito en la Red.
